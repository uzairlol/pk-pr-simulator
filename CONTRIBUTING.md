# Contributing to pk-pr-simulator
Thank you for your interest in contributing to this project! We welcome code contributions, bug reports, and feature suggestions.
## How to Contribute
### 1. Report Bugs & Request Features
* Search existing Github Issues before opening a new one.
* Use a clear, descriptive title and provide a minimal reproducible example if reporting a bug.
### 2. Development Process
1. Fork the repository and create your branch from `main`.
2. Ensure your folder structure follows the established `src/` layout.
3. Write clean, modular code and add tests for new functionality.
### 3. Pull Request Guidelines
* Adhere to standard Python linting configurations.
* Update documentation files inside `docs/` if modifying features.
* Open a PR with a clear description of the changes introduced.
